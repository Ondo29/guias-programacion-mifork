1. ¿ Cuales son las 4 caracteristicas básicas de la programación orientada a objetos? Describe brevemene cada una.

Encapsulamiento

Consiste en ocultar los detalles internos de un objeto y exponer solo lo necesario.

Permite que los datos y métodos estén protegidos, evitando que se acceda o modifique directamente desde fuera del objeto.

Ejemplo: una clase CuentaBancaria tiene un saldo privado, y solo permite cambiarlo mediante métodos como depositar() o retirar().

2️⃣ Abstracción

Es la representación simplificada de un objeto, mostrando solo lo relevante para el uso.

Permite centrarse en qué hace un objeto, sin preocuparse por cómo lo hace internamente.

Ejemplo: un Auto tiene métodos como acelerar() y frenar(), sin necesidad de conocer cómo funciona el motor internamente.

3️⃣ Herencia

Permite que una clase (hija) herede propiedades y métodos de otra clase (padre).

Favorece la reutilización de código y la creación de jerarquías de clases.

Ejemplo: Vehiculo puede ser la clase padre y Auto y Moto clases hijas que heredan atributos como color o velocidad.

4️⃣ Polimorfismo

Significa “muchas formas”, y permite que un mismo método se comporte de manera distinta según el objeto que lo use.

Facilita la flexibilidad y la adaptación de funciones a distintos tipos de objetos.

Ejemplo: un método dibujar() puede funcionar diferente para un objeto Circulo que para un Rectangulo.

4. ¿ Qué tres elemenetos definen a un objeto en programción orientada a objetos?

En Programación Orientada a Objetos (POO), un objeto se define por tres elementos:

Atributos → sus datos o características (estado).

6. ¿ Donde se almacenan en memoria los objetos? ¿ Es igual en todos los lenguajes? ¿ Qué es la recolección de basura?

¿Dónde se almacenan los objetos en memoria?

Generalmente, los objetos se almacenan en el heap (montículo).

El stack (pila) suele guardar variables locales y referencias.

El heap guarda los objetos creados dinámicamente.

🔹 ¿Es igual en todos los lenguajes?

No.

Depende del lenguaje y su modelo de memoria:

En Java → los objetos van al heap y el sistema maneja la memoria automáticamente.

En C++ → pueden estar en stack o heap, y el programador puede controlar la memoria.

Cada lenguaje tiene reglas distintas.

🔹 ¿Qué es la recolección de basura?

La recolección de basura (Garbage Collection) es un mecanismo automático que:

Detecta objetos que ya no se usan.

Libera la memoria que ocupaban.

Lenguajes como Java y Python la incluyen automáticamente.

13. Añade ahora otro nuevo método que se llame distanciaA, que reciba un punto como parámetro y calcule la distancia entre this y el punto proporcionado.

class Punto {
    int x;
    int y;

    // Constructor
    Punto(int x, int y) {
        this.x = x;
        this.y = y;
    }

    // Distancia al origen (0,0)
    double calculaDistanciaA0Origen() {
        return Math.sqrt(this.x * this.x + this.y * this.y);
    }

    // Distancia a otro punto
    double distanciaA(Punto otro) {
        int dx = this.x - otro.x;
        int dy = this.y - otro.y;
        return Math.sqrt(dx * dx + dy * dy);
    }
}

16. Reflexiona: ¿ Una clase es como un struct en C? ¿ Qué le falta al struct para ser como una clase y variables de ese tipo ser instancias?

Clase vs struct en C

Clase (Java, C++):

Contiene atributos (datos) y métodos (funciones) que definen el comportamiento.

Puede tener constructores para inicializar objetos.

Permite encapsulamiento (control de visibilidad: public, private, protected).

Las variables de tipo clase son instancias u objetos con identidad propia en memoria.

Puede tener herencia y polimorfismo.

struct en C:

Solo contiene datos, no métodos.

No tiene constructores ni destructores.

No controla la visibilidad de sus campos (todo es público).

Las variables de tipo struct se pueden copiar directamente, pero no tienen “comportamiento” asociado.

2. Qué le falta al struct para ser como una clase

Métodos → para definir comportamiento.

Constructores/destructores → para inicialización y limpieza.

Encapsulamiento → para proteger datos con private y public.

Herencia y polimorfismo → para reutilizar y extender estructuras.

Con esto, un struct podría convertirse en una clase completa, y las variables serían instancias u objetos en vez de simples bloques de datos.

17. Quitemos un poco de magia a todo esto: ¿ como se podría " emular", con struct en C, la clase punto, con su función para calcular la distncia al origen?
¿ Qué ha pasado con this?

1. Emulando la clase Punto en C

En Java tenemos esto:

class Punto {
    int x, y;
    double calculaDistanciaA0Origen() { ... }
}

En C no hay métodos dentro del struct, pero podemos hacer una función que reciba un puntero al struct, como si fuera this.

#include <stdio.h>
#include <math.h>

typedef struct {
    int x;
    int y;
} Punto;

// Función que “actúa como método”
double calculaDistanciaA0Origen(Punto *this) {
    return sqrt(this->x * this->x + this->y * this->y);
}

int main() {
    Punto p = {3, 4}; // "Constructor" manual
    printf("Distancia al origen: %.2f\n", calculaDistanciaA0Origen(&p));
    return 0;
}

Explicación clave:

Punto *this → en C no existe this, pero pasamos un puntero al objeto, que juega el mismo rol.

this->x y this->y → accedemos a los campos del “objeto”.

No hay encapsulamiento ni constructor automático, lo hacemos manualmente: Punto p = {3, 4};.

2. Qué ha pasado con this

En C no hay this porque no hay métodos dentro del struct.

Podemos simularlo pasando un puntero al struct a la función.

Cada función que “trabaja con un struct” recibe este puntero y puede acceder a los campos, exactamente como this en Java.

Resumen conceptual:

Java	C (simulado)
class Punto { int x,y; ...}	typedef struct { int x,y; } Punto;
double calculaDistanciaA0Origen()	double calculaDistanciaA0Origen(Punto *this)
this.x	this->x
Punto p = new Punto(3,4);	Punto p = {3,4};

Si quieres, puedo hacer la versión completa de Punto en C con función para distancia a otro punto también, imitando distanciaA(Punto otro) de Java. Esto deja muy claro la equivalencia.
